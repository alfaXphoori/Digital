# Chapter 4: แผนผังคาร์โนห์

## Karnaugh Maps (K-Maps)

---

## 4.1 บทนำ

**Karnaugh Map (K-Map)** คือเครื่องมือกราฟิกสำหรับลดรูปสมการบูลีนอย่างเป็นระบบ ให้ได้สมการที่สั้นที่สุด → สร้างวงจรด้วยเกตน้อยที่สุด

- ใช้ได้กับ **2–4 ตัวแปร** (สูงสุด 5–6 แต่ไม่สะดวก)
- ลำดับตัวแปรใช้ **Gray Code** (เปลี่ยนทีละ 1 บิต)

---

## 4.2 K-Map 2 ตัวแปร (4 เซลล์)

$$F(A,B) = \sum m(1, 2, 3)$$

```
          B=0    B=1
        ┌──────┬──────┐
  A=0   │ m0=0 │ m1=1 │
        ├──────┼──────┤
  A=1   │ m2=1 │ m3=1 │
        └──────┴──────┘
```

จัดกลุ่ม:
- กลุ่ม 1 (คอลัมน์ B=1): m1, m3 → $B$
- กลุ่ม 2 (แถว A=1): m2, m3 → $A$

$$F = A + B$$

---

## 4.3 K-Map 3 ตัวแปร (8 เซลล์)

$$F(A,B,C) = \sum m(0, 2, 5, 7)$$

```
            BC
   AB │ 00  │ 01  │ 11  │ 10  │
  ────┼─────┼─────┼─────┼─────┤
   0  │  1  │  0  │  0  │  1  │   A=0: m0, m1, m3, m2
  ────┼─────┼─────┼─────┼─────┤
   1  │  0  │  1  │  1  │  0  │   A=1: m4, m5, m7, m6
  ────┴─────┴─────┴─────┴─────┘
```

> ⚠️ **ลำดับคอลัมน์เป็น Gray Code:** 00, 01, 11, 10 (ไม่ใช่ 00, 01, 10, 11!)

จัดกลุ่ม:
- กลุ่ม 1: m0, m2 (คอลัมน์ 00 และ 10) → $\overline{A}\,\overline{C}$
- กลุ่ม 2: m5, m7 → $AC$

$$F = \overline{A}\,\overline{C} + AC$$

---

## 4.4 K-Map 4 ตัวแปร (16 เซลล์) ⭐

$$F(A,B,C,D) = \sum m(0, 1, 2, 5, 8, 9, 10)$$

```
            CD
  AB  │ 00  │ 01  │ 11  │ 10  │
  ────┼─────┼─────┼─────┼─────┤
  00  │  1  │  1  │  0  │  1  │   m0   m1   m3   m2
  ────┼─────┼─────┼─────┼─────┤
  01  │  0  │  1  │  0  │  0  │   m4   m5   m7   m6
  ────┼─────┼─────┼─────┼─────┤
  11  │  0  │  0  │  0  │  0  │   m12  m13  m15  m14
  ────┼─────┼─────┼─────┼─────┤
  10  │  1  │  1  │  0  │  1  │   m8   m9   m11  m10
  ────┴─────┴─────┴─────┴─────┘
```

จัดกลุ่ม:
- กลุ่ม 4 ช่อง: m0, m2, m8, m10 → $\overline{B}\,\overline{D}$
- กลุ่ม 4 ช่อง: m0, m1, m8, m9 → $\overline{B}\,\overline{C}$
- กลุ่ม 2 ช่อง: m1, m5 → $\overline{C}\,D$... (ตรวจสอบ: $\overline{A}\,\overline{C}\,D$ ?)

> 💡 ต้องตรวจสอบให้ครอบคลุม `1` ทุกช่อง

$$F = \overline{B}\,\overline{D} + \overline{B}\,\overline{C} + \overline{A}\,\overline{C}\,D$$

---

## 4.5 กฎการจัดกลุ่ม (Grouping Rules) ⭐

| ข้อ | กฎ |
|:---:|:---|
| 1 | ขนาดกลุ่มต้องเป็น **$2^n$** เท่านั้น: 1, 2, 4, 8, 16 |
| 2 | รูปร่างต้องเป็น **สี่เหลี่ยมผืนผ้า** (รวมถึง wrap-around) |
| 3 | กลุ่ม **ยิ่งใหญ่ยิ่งดี** (ลดรูปได้มาก) |
| 4 | ทุกช่อง `1` ต้องอยู่ใน **อย่างน้อย 1 กลุ่ม** |
| 5 | ช่อง `1` สามารถ **ซ้อนทับกลุ่ม** ได้ |
| 6 | จำนวนกลุ่มรวมให้ **น้อยที่สุด** |

### กลุ่มขนาด n → ลดได้กี่ตัวแปร?

| ขนาดกลุ่ม | ตัวแปรที่เหลือ (4-var K-Map) |
|:---:|:---:|
| 1 ช่อง | 4 ตัวแปร |
| 2 ช่อง | 3 ตัวแปร |
| 4 ช่อง | 2 ตัวแปร |
| 8 ช่อง | 1 ตัวแปร |
| 16 ช่อง | 0 ตัวแปร (= 1) |

## 4.6 Wrap-Around (ขอบเชื่อมกัน)

ใน K-Map 4 ตัวแปร — **ขอบซ้าย-ขวา** และ **บน-ล่าง** เชื่อมถึงกัน!

```
K-Map 4 ตัวแปร:
┌────┬────┬────┬────┐
│ m0 │ m1 │ m3 │ m2 │ ← เชื่อมกับแถว m8-m10
├────┼────┼────┼────┤
│ m4 │ m5 │ m7 │ m6 │
├────┼────┼────┼────┤
│ m12│ m13│ m15│ m14│
├────┼────┼────┼────┤
│ m8 │ m9 │ m11│ m10│ ← เชื่อมกับแถว m0-m2
└────┴────┴────┴────┘
  ↕                ↕
ซ้าย             ขวา เชื่อมกัน

ตัวอย่างกลุ่มที่เป็นไปได้:
• m0, m2, m8, m10 (4 มุม) ✅
• m0, m4, m12, m8 (คอลัมน์ซ้าย + ขวา) ✅  ← ผิด ต้องเป็นคอลัมน์เดียวกัน
• m0, m8 (แถวบน-ล่าง คอลัมน์เดียวกัน) ✅
```

**4 มุมรวมเป็น 1 กลุ่มได้!**

```
┌────┬────┬────┬────┐
│ ★  │    │    │ ★  │
├────┼────┼────┼────┤
│    │    │    │    │
├────┼────┼────┼────┤
│    │    │    │    │
├────┼────┼────┼────┤
│ ★  │    │    │ ★  │
└────┴────┴────┴────┘

m0, m2, m8, m10 → B̄D̄
```

## 4.7 Don't Care Conditions (สภาวะไม่แคร์)

เมื่อบาง input combination **ไม่มีโอกาสเกิดขึ้นจริง** หรือ **ไม่สนใจเอาต์พุต** สามารถกำหนดเป็น **`X`** (don't care)

$$F(A,B,C,D) = \sum m(1, 3, 7, 11, 15) + \sum d(0, 2, 5)$$

### กฎการใช้ Don't Care:

1. ช่อง `X` สามารถเลือกเป็น `0` หรือ `1` ตามสะดวก
2. เลือกเป็น `1` เมื่อทำให้กลุ่มใหญ่ขึ้น → ลดรูปได้มากขึ้น
3. เลือกเป็น `0` เมื่อไม่ช่วยอะไร
4. **ห้าม** สร้างกลุ่มที่มีแต่ `X` อย่างเดียว

**ตัวอย่าง:** BCD มีค่าเพียง 0–9 ดังนั้นค่า 10–15 เป็น don't care

---

## 4.8 POS จาก K-Map

### ขั้นตอน:

1. จัดกลุ่มช่อง **`0`** แทน (ไม่ใช่ `1`)
2. เขียน complement ของแต่ละกลุ่ม → ได้ maxterm
3. นำ maxterm มาคูณกัน → ได้ POS

**ตัวอย่าง:** $F(A,B) = \sum m(1,2,3)$

```
        B=0   B=1
      ┌─────┬─────┐
A=0   │  0  │  1  │
      ├─────┼─────┤
A=1   │  1  │  1  │
      └─────┴─────┘
```

ช่อง `0` อยู่ที่ m0 เท่านั้น → $\overline{F} = \overline{A}\,\overline{B}$ → $F = A + B$

---

## 4.9 ขั้นตอนสรุปการใช้ K-Map

```
1. สร้างตาราง K-Map ตามจำนวนตัวแปร (ลำดับ Gray Code)
          ↓
2. ใส่ค่า 1, 0, X ลงในเซลล์ตามฟังก์ชัน
          ↓
3. จัดกลุ่ม 1 (SOP) หรือ 0 (POS)
   - กลุ่มใหญ่สุด, จำนวนน้อยสุด
   - สี่เหลี่ยม, ขนาด 2ⁿ
   - ขอบเชื่อมกัน, ซ้อนทับได้
   - Don't care ช่วยขยายกลุ่ม
          ↓
4. อ่านตัวแปรจากแต่ละกลุ่ม
   - ตัวแปรที่ไม่เปลี่ยน = เอาไว้
   - ตัวแปรที่เปลี่ยน = ตัดออก
          ↓
5. เขียนสมการ SOP หรือ POS
```

## 4.10 เปรียบเทียบวิธีลดรูป

| เกณฑ์ | Boolean Algebra | K-Map |
|:---|:---:|:---:|
| จำนวนตัวแปร | ไม่จำกัด | 2–6 |
| ความเป็นระบบ | ต้องใช้ประสบการณ์ | เป็นขั้นตอนชัดเจน |
| ผลลัพธ์ดีที่สุด? | ไม่แน่ | ใช่ (สำหรับ 2-level) ✅ |
| เหมาะสำหรับ | ลดรูปทั่วไป | ลดรูป SOP/POS |

## แบบฝึกหัดท้ายบท

1. ลดรูป $F(A,B,C) = \sum m(0, 1, 2, 3, 7)$ โดยใช้ K-Map (SOP)
2. ลดรูป $F(A,B,C,D) = \sum m(0, 1, 3, 5, 7, 8, 9, 11, 15)$ (SOP)
3. ลดรูป $F(A,B,C,D) = \sum m(1, 5, 6, 7, 11, 12, 13, 15) + \sum d(3, 9, 10)$
4. หา POS ของ $F(A,B,C) = \sum m(1, 3, 5, 7)$
5. ออกแบบวงจรจากผลลัพธ์ K-Map ข้อ 2 โดยใช้เกต NAND เท่านั้น
