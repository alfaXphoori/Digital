# Chapter 8: หน่วยความจำและอุปกรณ์ลอจิกโปรแกรมได้

## Memory Devices & Programmable Logic (PLD)

---

## 8.1 บทนำ (Introduction)

ในระบบดิจิทัลและคอมพิวเตอร์ **หน่วยความจำ (Memory)** ทำหน้าที่จัดเก็บข้อมูล (Data) และชุดคำสั่ง (Instruction) เพื่อให้หน่วยประมวลผลนำไปใช้งาน ส่วน **อุปกรณ์ลอจิกโปรแกรมได้ (PLD)** เป็นอุปกรณ์ที่ช่วยให้เราสามารถสร้างวงจรลอจิกที่ซับซ้อนลงในชิปเพียงตัวเดียวได้ โดยการกำหนดฟังก์ชันการทำงานผ่านซอฟต์แวร์ แทนการต่อเกตจำนวนมากบนแผงวงจร

---

## 8.2 สถาปัตยกรรมหน่วยความจำ (Memory Architecture)

โครงสร้างภายในของหน่วยความจำประกอบด้วยเซลล์จัดเก็บข้อมูลที่เรียงตัวกันเป็นแถว (Rows) และคอลัมน์ (Columns) โดยมีส่วนประกอบหลักดังนี้:

```text
              Address Bus (n bits)
                    │ [ระบุตำแหน่ง]
                    ↓
            ┌───────────────┐
            │   Address     │ 
            │   Decoder     │ [เลือกแถวที่ตรงกับที่อยู่]
            │  (n → 2ⁿ)     │
            ├───────────────┤
  CS ──────→│               │
  R/W̄ ─────→│  Memory Array │   ความจุรวม = 2ⁿ x m bits
  OE ──────→│  (Storage)    │   (n แถว, m คอลัมน์)
            │               │
            ├───────────────┤
            │  I/O Buffers  │ [พักข้อมูลชั่วคราว]
            └───────┬───────┘
                    │ [รับ/ส่งข้อมูล]
              Data Bus (m bits)
```

### สัญญาณควบคุมที่สำคัญ (Control Signals)
1. **Address Bus:** เส้นสัญญาณระบุตำแหน่งข้อมูล (Input)
2. **Data Bus:** เส้นสัญญาณรับ-ส่งข้อมูล (Bi-directional)
3. **CS (Chip Select) / CE (Chip Enable):** เปิดการใช้งานชิปตัวนั้นๆ
4. **R/W̄ (Read/Write):** กำหนดทิศทางข้อมูล (1 = อ่าน, 0 = เขียน)
5. **OE (Output Enable):** เปิดเอาต์พุตของชิปให้ต่อกับ Data Bus (Tri-state control)

### การคำนวณความจุ (Memory Capacity)
ความจุวัดเป็น **จำนวนตำแหน่ง (Address) x ความกว้างข้อมูล (Bit per Address)**
- $2^{10} = 1024 = 1K$
- $2^{20} = 1,048,576 = 1M$

**ตัวอย่าง:** หน่วยความจำที่มี Address 12 เส้น และ Data 8 เส้น
- จำนวนตำแหน่ง = $2^{12} = 4096 = 4K$
- ความกว้างข้อมูล = 8 บิต (1 ไบต์)
- ความจุรวม = 4K x 8 bits = **4 KB**

## 8.3 ประเภทของหน่วยความจำ (Memory Types)

แบ่งตามลักษณะการเก็บข้อมูลเมื่อไม่มีกระแสไฟฟ้า:

### 8.3.1 RAM (Random Access Memory) - หน่วยความจำแบบลบเลือนได้ (Volatile)
ข้อมูลจะหายไปเมื่อปิดเครื่อง อ่านและเขียนข้อมูลได้รวดเร็วเท่ากันทุกตำแหน่ง

| คุณสมบัติ | SRAM (Static RAM) | DRAM (Dynamic RAM) |
|:---|:---:|:---:|
| **องค์ประกอบหลัก** | Flip-Flop (6 Transistors) | Capacitor (1T-1C) |
| **ความเร็ว** | **สูงมาก** ✅ | ช้ากว่า |
| **ความหนาแน่น** | ต่ำ (ชิปใหญ่) | **สูงมาก** ✅ |
| **การ Refresh** | ไม่ต้อง | **ต้องทำตลอดเวลา** (เพราะประจุรั่ว) |
| **ราคา** | แพง | ถูกกว่า |
| **การใช้งาน** | Cache Memory, Register | Main Memory (DDR4, DDR5) |

### 8.3.2 ROM (Read-Only Memory) - หน่วยความจำแบบไม่ลบเลือน (Non-Volatile)
ข้อมูลไม่หายเมื่อปิดเครื่อง เดิมทีออกแบบมาเพื่ออ่านอย่างเดียว แต่ปัจจุบันพัฒนาให้เขียนใหม่ได้หลายวิธี

1. **Mask ROM:** บันทึกข้อมูลจากโรงงาน (แก้ไขไม่ได้)
2. **PROM (Programmable ROM):** เขียนได้ครั้งเดียวโดยผู้ใช้
3. **EPROM (Erasable PROM):** ลบข้อมูลได้ด้วยแสง UV (สังเกตได้จากช่องกระจกใสบนชิป)
4. **EEPROM (Electrically Erasable PROM):** ลบและเขียนใหม่ได้ด้วยไฟฟ้า (ทีละไบต์)
5. **Flash Memory:** พัฒนาจาก EEPROM ให้ลบข้อมูลเป็นบล็อก (Block-level) ได้รวดเร็วและทนทาน

---

## 8.4 NAND Flash vs NOR Flash (ความรู้ขั้นสูง)

ปัจจุบัน Flash Memory แบ่งออกเป็น 2 สถาปัตยกรรมหลักตามลักษณะการต่อเซลล์:

| คุณสมบัติ | NAND Flash | NOR Flash |
|:---|:---:|:---:|
| **ความจุ / ราคา** | สูง / ถูก ✅ | ต่ำ / แพง |
| **ความเร็วการอ่าน** | ช้ากว่า (อ่านเป็นบล็อก) | **เร็วมาก** (สุ่มตำแหน่งได้) ✅ |
| **ความเร็วการเขียน** | **เร็ว** ✅ | ช้า |
| **ความทนทาน** | สูง (หลักแสนครั้ง) | ต่ำกว่า |
| **การนำไปใช้** | SSD, USB Drive, SD Card | เก็บ BIOS, Firmware |

## 8.5 ไทม์มิ่งของหน่วยความจำ (Memory Timing)

ในการออกแบบระบบจริง เราต้องพิจารณาเวลาที่สัญญาณใช้ในการตอบสนอง:
- **Access Time ($t_{AA}$):** เวลาที่นับตั้งแต่ป้อนที่อยู่ (Address) จนกระทั่งข้อมูลปรากฏบน Data Bus อย่างเสถียร
- **Write Cycle Time ($t_{WC}$):** เวลาขั้นต่ำที่ต้องรอเพื่อให้กระบวนการเขียนข้อมูลเสร็จสมบูรณ์ก่อนจะเริ่มคำสั่งถัดไป

## 8.6 การถอดรหัสที่อยู่ (Address Decoding)

ในระบบคอมพิวเตอร์มักมีชิปหน่วยความจำหลายตัวต่อรวมกันบนบัสเดียว เราจึงต้องมีวงจร **Address Decoder** เพื่อเลือกชิปที่ต้องการ (Chip Select)

### 1. Full Decoding
ใช้เส้น Address ที่เหลือทั้งหมดมาถอดรหัส ทำให้ชิปแต่ละตัวมีตำแหน่งที่ไม่ซ้ำซ้อนกันเลย

### 2. Partial Decoding (การถอดรหัสบางส่วน)
ใช้เส้น Address เพียงบางส่วนเพื่อประหยัดวงจร แต่จะเกิดปรากฏการณ์ **Mirroring** (ข้อมูลชุดเดียวโผล่มาในหลายๆ ที่อยู่)
**ตัวอย่าง:** ระบบ 16-bit Address (64KB Space) ใช้ชิป 8KB สองตัว
- ชิป 1 (ROM): อยู่ที่ตำแหน่ง 0000h - 1FFFh
- ชิป 2 (RAM): อยู่ที่ตำแหน่ง 2000h - 3FFFh

#### ตารางการถอดรหัสที่อยู่ (Address Decoding Truth Table)

| A15 | A14 | A13 | ช่วงที่อยู่ (Hex) | ชิปที่ถูกเลือก | สัญญาณเปิดใช้งาน |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 0 | 0 | 0 | 0000h - 1FFFh | **ROM** | CS1 (Active LOW) |
| 0 | 0 | 1 | 2000h - 3FFFh | **RAM** | CS2 (Active LOW) |
| 0 | 1 | 0 | 4000h - 5FFFh | Unused | - |
| ... | ... | ... | ... | ... | ... |

> 💡 **ใช้อุปกรณ์ 74138 (Decoder) ในการถอดรหัสบิต A15, A14, A13** เพื่อสร้างสัญญาณ Chip Select (CS) ให้กับหน่วยความจำแต่ละตัว

---

## 8.7 อุปกรณ์ลอจิกโปรแกรมได้ (Programmable Logic Devices - PLD)

PLD คือชิปที่เราสามารถ "โปรแกรม" ความเชื่อมต่อภายในได้ โครงสร้างหลักประกอบด้วย **AND Array** และ **OR Array**

### 1. PLA (Programmable Logic Array)
- **AND Array:** โปรแกรมได้
- **OR Array:** โปรแกรมได้
- **ข้อดี:** ยืดหยุ่นสูงสุด ใช้ทรัพยากรน้อย
- **ข้อเสีย:** ความเร็วต่ำ (เพราะสัญญาณผ่านส่วนที่โปรแกรมได้ 2 ระดับ)

#### 8.7.1 โครงสร้างลอจิกเกตภายในของ PLA (Internal Logic Gate Structure)

เพื่อให้เข้าใจว่าจุดตัด (X) ใน Matrix กลายเป็นวงจรไฟฟ้าได้อย่างไร ให้พิจารณาโครงสร้างที่ประกอบด้วย **AND-OR Gates** ดังนี้:

**1. ส่วนอินพุต (Input Buffers/Inverters):**
ทุกอินพุตจะถูกแยกออกเป็น 2 เส้น คือสัญญาณปกติ ($A$) และสัญญาณกลับค่า ($\bar{A}$) เพื่อให้พร้อมสำหรับการสร้าง Product Term ทุกรูปแบบ

**2. ส่วนสร้างเทอมคูณ (Programmable AND Array):**
ใช้ AND Gate ขนาดใหญ่ที่มีจำนวนขาเท่ากับ $2 \times \text{จำนวน Input}$ โดยมีฟิวส์ (Fuse) เชื่อมต่ออยู่ทุกจุด เมื่อเรา "โปรแกรม" เราจะทำการเผาฟิวส์ในจุดที่ไม่ต้องการออก ให้เหลือเฉพาะตัวแปรที่ต้องการคูณกัน

**3. ส่วนสร้างเทอมบวก (Programmable OR Array):**
รับผลลัพธ์จาก AND Gate (เรียกว่า Product Terms) มาผ่าน OR Gate ที่สามารถเลือกเชื่อมต่อได้เช่นกัน เพื่อรวมเทอมคูณเข้าด้วยกันตามสมการ Sum of Products (SOP)

---

**ตัวอย่าง: การสร้างวงจรจากสมการด้วย PLA**

สมมติเราต้องการสร้างวงจรที่มี 2 เอาต์พุต ดังนี้:
- $O_0 = A \cdot B + \bar{A} \cdot \bar{B}$
- $O_1 = A \cdot B + A \cdot \bar{B}$

**แผนผังวงจรระดับเกต (Logic Gate Diagram):**

```text
อินพุต (Inputs)           AND Array (เทอมคูณ)             OR Array (เทอมบวก)
  A      B
  │ ┌────┼────┐ 
  │ │    │    │          ┌──────────┐
  A A'   B    B'         │ AND Gate │ P1 (A·B)
  │ │    │    │   ───────┤ (Fuse)   ├──────────┬───────────┐
  X─┼────X────┼──────────┤          │          │           │
  │ │    │    │          └──────────┘          │     ┌─────┴─────┐
  │ │    │    │          ┌──────────┐          X     │  OR Gate  ├──► O0
  │ │    │    │          │ AND Gate │ P2 (A'·B') │     │  (Fuse)   │
  ┼─X────┼────X──────────┤ (Fuse)   ├──────────┼────►│           │
  │ │    │    │          │          │          X     └───────────┘
  │ │    │    │          └──────────┘          │
  │ │    │    │          ┌──────────┐          │     ┌─────┴─────┐
  │ │    │    │          │ AND Gate │ P3 (A·B')│     │  OR Gate  ├──► O1
  X─┼────┼────X──────────┤ (Fuse)   ├──────────┴────►│  (Fuse)   │
  │ │    │    │          │          │                │           │
  │ │    │    │          └──────────┘                └───────────┘
```

**ขั้นตอนการทำงาน:**
1.  **P1** เชื่อมต่อกับ $A$ และ $B$ $\rightarrow$ ได้ $A \cdot B$
2.  **P2** เชื่อมต่อกับ $\bar{A}$ และ $\bar{B}$ $\rightarrow$ ได้ $\bar{A} \cdot \bar{B}$
3.  **P3** เชื่อมต่อกับ $A$ และ $\bar{B}$ $\rightarrow$ ได้ $A \cdot \bar{B}$
4.  **เอาต์พุต $O_0$** เลือกรับสัญญาณจาก $P_1$ และ $P_2$ มา OR กัน $\rightarrow$ $A \cdot B + \bar{A} \cdot \bar{B}$
5.  **เอาต์พุต $O_1$** เลือกรับสัญญาณจาก $P_1$ และ $P_3$ มา OR กัน $\rightarrow$ $A \cdot B + A \cdot \bar{B}$ (สังเกตว่ามีการแชร์ $P_1$ ร่วมกันได้)

---

### 2. PAL (Programmable Array Logic)
- **AND Array:** โปรแกรมได้
- **OR Array:** **Fixed (กำหนดมาตายตัว)**
- **ข้อดี:** **เร็วมาก** ✅ เพราะลดส่วนที่โปรแกรมได้ลง ออกแบบวงจรง่าย
- **ข้อเสีย:** ยืดหยุ่นน้อยกว่า PLA

### 3. GAL (Generic Array Logic)
- พัฒนาจาก PAL โดยใช้เทคโนโลยี EEPROM ทำให้ลบและเขียนใหม่ได้หลายครั้ง และมี Output Logic Macrocell (OLMC) ที่ปรับเปลี่ยนรูปแบบเอาต์พุตได้

---

## 8.8 ตัวอย่างการออกแบบ: วงจรควบคุมไมโครเวฟด้วย PAL (Smart Microwave Controller)

เพื่อให้เห็นภาพการใช้งาน **PAL (Programmable Array Logic)** อย่างละเอียด เราจะออกแบบวงจรควบคุมไมโครเวฟ จุดเด่นของ PAL คือ **AND Array โปรแกรมได้ แต่ OR Array ถูกกำหนดมาตายตัว** ทำให้วงจรเร็วกว่า PLA แต่มักจะต้อง "สร้าง Product Term ซ้ำ" หากเอาต์พุตหลายตัวใช้เงื่อนไขเดียวกัน

### 1. กำหนดตัวแปร (I/O Mapping)

| ประเภท | สัญลักษณ์ | ความหมาย (1 = Active) |
|:---:|:---:|:---|
| **Input** | **D** (Door) | ประตูปิดสนิท |
| | **T** (Timer) | ยังมีเวลาเหลือ |
| | **W** (Weight) | มีอาหารวางบนจานหมุน |
| | **S** (Start) | กดปุ่มเริ่มทำงาน |
| | **H** (Overheat)| เครื่องร้อนเกินไป |
| **Output**| **M** (Magnetron)| หัวคลื่นทำงาน (เวฟอาหาร) |
| | **F** (Fan) | พัดลมระบายอากาศ |
| | **L** (Light) | ไฟส่องสว่างภายใน |
| | **B** (Buzzer) | เสียงเตือน |

### 2. สมการลอจิก (Boolean Equations)

ในโครงสร้าง PAL เรามักเขียนในรูป **Sum of Products (SOP)**:
- $M = D \cdot T \cdot W \cdot S \cdot \bar{H}$ (เครื่องทำงานเมื่อประตูปิด, มีเวลา, มีอาหาร, กดเริ่ม และไม่ร้อนจัด)
- $F = (D \cdot T \cdot W \cdot S \cdot \bar{H}) + H$ (พัดลมหมุนเมื่อเครื่องทำงาน **หรือ** เครื่องร้อนเกิน)
- $L = \bar{D} + (D \cdot T \cdot W \cdot S \cdot \bar{H})$ (ไฟสว่างเมื่อเปิดประตู **หรือ** เครื่องทำงาน)
- $B = (\bar{T} \cdot D) + H$ (ดังเมื่อหมดเวลาขณะประตูปิด **หรือ** เครื่องร้อน)

### 3. โครงสร้างภายใน Matrix ของ PAL (Logic Implementation)

ใน PAL เราต้องโปรแกรมส่วน **AND Array** เพื่อสร้าง Product Terms (P1-P8) และต่อเข้ากับ **Fixed OR** ที่กำหนดไว้แล้ว (สมมติให้ OR gate แต่ละตัวรับได้ 2 อินพุตตายตัว)

```text
[ AND Array - โปรแกรมได้ ]
อินพุต (Inputs)       │ P1 │ P2 │ P3 │ P4 │ P5 │ P6 │ P7 │
──────────────────────┼────┼────┼────┼────┼────┼────┼────┤
D   (Door)            │ X  │ X  │ ·  │ ·  │ X  │ X  │ ·  │
D'  (NOT Door)        │ ·  │ ·  │ ·  │ X  │ ·  │ ·  │ ·  │
T   (Timer)           │ X  │ X  │ ·  │ ·  │ X  │ ·  │ ·  │
T'  (NOT Timer)       │ ·  │ ·  │ ·  │ ·  │ ·  │ X  │ ·  │
W   (Weight)          │ X  │ X  │ ·  │ ·  │ X  │ ·  │ ·  │
W'  (NOT Weight)      │ ·  │ ·  │ ·  │ ·  │ ·  │ ·  │ ·  │
S   (Start)           │ X  │ X  │ ·  │ ·  │ X  │ ·  │ ·  │
S'  (NOT Start)       │ ·  │ ·  │ ·  │ ·  │ ·  │ ·  │ ·  │
H   (Overheat)        │ ·  │ ·  │ X  │ ·  │ ·  │ ·  │ X  │
H'  (NOT Overheat)    │ X  │ X  │ ·  │ ·  │ X  │ ·  │ ·  │
──────────────────────┼────┼────┼────┼────┼────┼────┼────┤
Product Term =        │D·T·│D·T·│ H  │ D' │D·T·│D·T'│ H  │
                      │W·S·│W·S·│    │    │W·S·│    │    │
                      │H'  │H'  │    │    │H'  │    │    │

[ OR Array - ตายตัว (Fixed) ]
P1       ─────────────────────────────────────────────► M (Magnetron)
P2 ──┐
P3 ──┴───────────────────────────────────────────────► F (Fan)
P4 ──┐
P5 ──┴───────────────────────────────────────────────► L (Light)
P6 ──┐
P7 ──┴───────────────────────────────────────────────► B (Buzzer)

(X = เชื่อมต่อ, · = ไม่เชื่อมต่อ)
```

> 💡 **ข้อสังเกตสำคัญของ PAL:** สังเกตว่า Product Term สำหรับเงื่อนไขเครื่องทำงาน ($D \cdot T \cdot W \cdot S \cdot \bar{H}$) **ต้องถูกสร้างซ้ำถึง 3 ครั้ง (P1, P2, P5)** เพราะโครงสร้าง OR Array ของ PAL ถูกตีกรอบตายตัว (Fixed) ไม่สามารถแชร์ Product Term ระหว่างเอาต์พุตได้เหมือน PLA แต่นี่คือสิ่งที่ทำให้ PAL ทำงานได้รวดเร็วกว่า PLA มากในฮาร์ดแวร์จริง

### 4. ตารางความจริงสรุปการทำงาน (Truth Table)

| สถานะ/เงื่อนไข | Magnetron (M) | Fan (F) | Light (L) | Buzzer (B) | หมายเหตุ |
|:---|:---:|:---:|:---:|:---:|:---|
| ประตูเปิด (D=0) | 0 | 0 | 1 | 0 | เครื่องหยุด, ไฟติด |
| ประตูปิด + หมดเวลา (D=1, T=0) | 0 | 0 | 0 | 1 | เสียงเตือนดัง |
| กำลังทำงานปกติ (D,T,W,S=1, H=0) | 1 | 1 | 1 | 0 | เวฟอาหาร, ไฟติด, พัดลมหมุน |
| เครื่องร้อนจัดขณะทำงาน (H=1) | 0 | 1 | 0 | 1 | หยุดเวฟ, พัดลมไล่ความร้อน, ไซเรนดัง |

## 8.9 เทคโนโลยีระดับสูง: CPLD และ FPGA

สำหรับงานที่ซับซ้อนมาก เช่น การสร้าง CPU หรือวงจรประมวลผล AI:

### CPLD (Complex PLD)
ประกอบด้วย PAL-like blocks หลายๆ ตัวเชื่อมต่อกัน เหมาะกับงานลอจิกเชิงผสมที่ซับซ้อน

### FPGA (Field Programmable Gate Array)
ประกอบด้วย **Logic Blocks (CLBs)** จำนวนมหาศาล และ **Interconnect** ที่ซับซ้อน
- ใช้โครงสร้าง **Look-Up Table (LUT)** แทนเกตจริง
- มีหน่วยความจำ RAM และวงจรคำนวณคณิตศาสตร์ภายใน
- โปรแกรมใหม่ได้ไม่จำกัดด้วยภาษา **VHDL** หรือ **Verilog**

## 8.10 การใช้งาน PLA ในระบบจริง: ระบบควบคุมลิฟต์ความเร็วสูง 4 ชั้น (Advanced 4-Floor Elevator)

ในส่วนนี้เราจะออกแบบ "มันสมอง" ของลิฟต์สำหรับอาคาร 4 ชั้น ซึ่งมีความซับซ้อนสูงกว่าปกติ โดยเพิ่มระบบควบคุมความเร็ว (High/Low Speed) เพื่อการจอดที่นุ่มนวล และระบบความปลอดภัยแบบหลายชั้น

### 1. การกำหนดอินพุตและเอาต์พุต (Consistent I/O Mapping)

**กลุ่มอินพุต (14 Inputs):**
- **Floor Sensors:** `Sen_F1` ถึง `Sen_F4` ($I_0 - I_3$)
- **Call Buttons:** `Btn_C1` ถึง `Btn_C4` ($I_4 - I_7$)
- **Safety & Status:** 
  - $I_8$: `Stat_Door` (ประตูปิดสนิท)
  - $I_9$: `Safe_Weight` (น้ำหนักเกิน)
  - $I_{10}$: `Safe_EMG` (ปุ่มหยุดฉุกเฉิน)
  - $I_{11}$: `Sen_Obs` (สิ่งกีดขวางประตู)
  - $I_{12}$: `Safe_Fire` (ตรวจจับไฟไหม้)
  - $I_{13}$: `Safe_Hot` (มอเตอร์ร้อนจัด)

**กลุ่มเอาต์พุต (6 Outputs):**
- $O_0$: `Mtr_Up` (ดึงขึ้น)
- $O_1$: `Mtr_Dn` (หย่อนลง)
- $O_2$: `Spd_High` (วิ่งเร็วข้ามชั้น)
- $O_3$: `Brk_Engage` (สั่งล็อคเบรก)
- $O_4$: `Door_Open` (สั่งเปิดประตู)
- $O_5$: `Alm_Siren` (ไซเรนเตือนภัย)

### 2. สมการตรรกะระดับสูง (Consistent Boolean Logic)

เพื่อให้มอเตอร์ทำงานได้อย่างปลอดภัย เราจะใช้เงื่อนไข **Safety_Ready ($S$)**:
$S = \text{Stat_Door} \cdot \overline{\text{Safe_Weight}} \cdot \overline{\text{Safe_EMG}} \cdot \overline{\text{Safe_Fire}} \cdot \overline{\text{Safe_Hot}}$

**สมการเอาต์พุต (SOP):**
- **Mtr_Up ($O_0$):**
  $O_0 = [(\text{Sen_F1} \cdot (\text{Btn_C2+C3+C4})) + (\text{Sen_F2} \cdot (\text{Btn_C3+C4})) + (\text{Sen_F3} \cdot \text{Btn_C4})] \cdot S$
- **Mtr_Dn ($O_1$):**
  $O_1 = [(\text{Sen_F4} \cdot (\text{Btn_C3+C2+C1})) + (\text{Sen_F3} \cdot (\text{Btn_C2+C1})) + (\text{Sen_F2} \cdot \text{Btn_C1})] \cdot S$
- **Spd_High ($O_2$):** วิ่งความเร็วสูงเมื่อต้องวิ่งข้ามมากกว่า 1 ชั้น
  $O_2 = [(\text{Sen_F1} \cdot (\text{Btn_C3+C4})) + (\text{Sen_F4} \cdot (\text{Btn_C2+C1}))] \cdot S$
- **Door_Open ($O_4$):**
  $O_4 = (\text{Sen_F1} \cdot \text{Btn_C1}) + (\text{Sen_F2} \cdot \text{Btn_C2}) + (\text{Sen_F3} \cdot \text{Btn_C3}) + (\text{Sen_F4} \cdot \text{Btn_C4}) + \text{Sen_Obs} + \text{Safe_Weight}$
- **Alm_Siren ($O_5$):**
  $O_5 = \text{Safe_Weight} + \text{Safe_EMG} + \text{Safe_Fire} + \text{Safe_Hot}$

#### 8.10.5 รายละเอียดการเชื่อมต่อฮาร์ดแวร์ (Hardware Wiring Details)

1.  **การเชื่อมต่อมอเตอร์หลัก (VFD Interface):**
    - เอาต์พุต **Move_Up ($O_0$)**, **Move_Down ($O_1$)**, และ **Speed_High ($O_2$)** จะต่อเข้ากับอินพุตดิจิทัลของ **Variable Frequency Drive (VFD)**
    - VFD จะทำหน้าที่เปลี่ยนความถี่ไฟฟ้าเพื่อควบคุมมอเตอร์ 3 เฟส ให้หมุนด้วยความเร็วและทิศทางที่ PLA กำหนด
2.  **ระบบเบรกความปลอดภัย (Brake Interlock):**
    - เอาต์พุต **Brk_Engage ($O_3$)** จะควบคุม Magnetic Contactor ที่จ่ายไฟให้คอยล์เบรกแม่เหล็กไฟฟ้า (ปกติเบรกจะจับค้างไว้ด้วยสปริง และจะคลายเมื่อมีไฟจ่ายเข้า $O_3$ เท่านั้น - Fail-safe)
3.  **การป้องกันสัญญาณรบกวน (Isolation):**
    - อินพุตทั้ง 14 ตัว จะต้องผ่าน **Opto-Isolators** ก่อนเข้า PLA เพื่อป้องกันแรงดันกระชาก (Surge) จากมอเตอร์ทำลายชิป
4.  **การจดจำสถานะ (Flip-Flop Storage):**
    - ใช้ **D-Flip Flops** เก็บสถานะปัจจุบัน (Present State) และส่งกลับเข้าเป็นอินพุตของ PLA เพื่อใช้ตัดสินใจในสถานะถัดไป (Next State)

### 4. จุดเด่นของการออกแบบระดับสูงนี้

1. **Redundancy Safety:** หากมอเตอร์ร้อนจัด (`Safe_Hot`) หรือเกิดไฟไหม้ (`Safe_Fire`) ตรรกะใน AND Array จะตัดการทำงานของมอเตอร์ทันทีโดยไม่ต้องผ่านซอฟต์แวร์
2. **Smooth Deceleration:** การแยกเอาต์พุต **Speed_High** ออกมา ช่วยให้ระบบขับเคลื่อนรู้ว่าควรชะลอความเร็วเมื่อเข้าใกล้ชั้นเป้าหมาย
3. **Logic Optimization:** การใช้ PLA ช่วยให้เราสามารถรวมเงื่อนไขความปลอดภัย 6 อย่างเข้าเป็น Product Term เดียว ($S$) แล้วนำไป "แชร์" ให้กับมอเตอร์ทุกตัวได้ ช่วยลดจำนวนเกตลงได้มหาศาล
