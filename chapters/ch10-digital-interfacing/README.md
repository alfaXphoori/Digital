# Chapter 10: การเชื่อมต่อระบบดิจิทัลและวงจรจำลอง

## Digital Interfacing: ADC, DAC & Tinkercad

---

## 10.1 บทนำ
ในโลกกายภาพ (Real World) ปรากฏการณ์ตามธรรมชาติเช่น อุณหภูมิ แสง เสียง หรือความดัน ล้วนเป็นสัญญาณ **แอนะล็อก (Analog)** ที่มีความต่อเนื่อง แต่ระบบคอมพิวเตอร์และลอจิกเกตทำงานด้วยสัญญาณ **ดิจิทัล (Digital)** ซึ่งมีเพียง 2 สถานะ (0 และ 1) 

ดังนั้น ระบบดิจิทัลที่สมบูรณ์จึงต้องมี "ตัวกลาง" เพื่อเชื่อมต่อโลกทั้งสองใบเข้าด้วยกัน ผ่านกระบวนการแปลงสัญญาณ (Conversion) และการปรับระดับทางไฟฟ้า (Electrical Interfacing)

```text
 Analog World                                Digital World
      │                                            │
      │   Sensor ───→ [ADC] ──────→ [Microcontroller / CPU / FPGA]
      │                                            │
      │ Actuator ←─── [DAC] ←───── [Processing Logic]
      │                                            │
```

## 10.2 DAC (Digital-to-Analog Converter)
**หลักการ:** แปลงรหัสดิจิทัล (Binary) ให้กลายเป็นแรงดันไฟฟ้าหรือกระแสไฟฟ้าแอนะล็อก

### สมการพื้นฐานของ DAC
$$V_{out} = V_{ref} \times \frac{D}{2^n}$$
- $V_{ref}$ = แรงดันไฟฟ้าอ้างอิงสูงสุด (Reference Voltage)
- $D$ = ค่าของข้อมูลดิจิทัลที่ป้อนเข้า (ฐานสิบ: 0 ถึง $2^n - 1$)
- $n$ = ความละเอียดของ DAC (จำนวนบิต)

### 10.2.1 พารามิเตอร์ที่สำคัญของ DAC
- **Resolution (ความละเอียด):** กำหนดโดยจำนวนบิต ($n$) ยิ่งบิตเยอะ ยิ่งแบ่งแรงดันได้ละเอียด (จำนวนขั้น = $2^n$)
- **Step Size (ขนาดของ 1 ขั้น) หรือ LSB Weight:** การเปลี่ยนแปลงของ $V_{out}$ เมื่ออินพุตเปลี่ยนไป 1 บิต ต่ำสุด ($LSB$)
  $$\text{Step Size} = \frac{V_{ref}}{2^n}$$
- **Full-Scale Voltage ($V_{FS}$):** แรงดันเอาต์พุตสูงสุดที่เป็นไปได้ จะมีค่าน้อยกว่า $V_{ref}$ อยู่ 1 Step เสมอ
  $$V_{FS} = V_{ref} - \text{Step Size} = V_{ref} \times \frac{2^n - 1}{2^n}$$

### 10.2.2 โครงสร้างวงจร DAC
มีวงจรพื้นฐานที่นิยมใช้ 2 รูปแบบ:

**1. Weighted Resistor DAC (วงจรตัวต้านทานถ่วงน้ำหนัก)**
ใช้ตัวต้านทานที่มีค่าเพิ่มขึ้นเป็น 2 เท่าตามลำดับบิต ($R, 2R, 4R, 8R, ...$) ต่อเข้ากับ Op-Amp
- **ข้อดี:** เข้าใจง่าย ตรงไปตรงมา
- **ข้อเสีย:** เมื่อจำนวนบิตสูงขึ้น ค่าความต้านทานจะแตกต่างกันมากเกินไป (เช่น บิตที่ 8 ต้องใช้ $128R$) ทำให้สร้างในไอซีได้ยากและขาดความแม่นยำ

**2. R-2R Ladder DAC (วงจรแบบบันได R-2R)**
แก้ปัญหาของแบบแรกโดยใช้ตัวต้านทานเพียง 2 ค่าเท่านั้น คือ $R$ และ $2R$ ต่อเป็นโครงข่ายบันได
- **ข้อดี:** สร้างลงในชิปไอซีได้ง่ายมาก เพราะตัวต้านทานมีแค่ 2 ค่า อัตราส่วนความคลาดเคลื่อนจึงต่ำ เป็นโครงสร้างที่นิยมใช้ที่สุดในปัจจุบัน

```text
[โครงสร้าง R-2R Ladder DAC (4-bit)]
 Vref
  │
  ├──[2R]──┬──[2R]──┬──[2R]──┬──[2R]──┬──→ GND
  │        │        │        │        │
 [R]      [R]      [R]      [R]      [R]
  │        │        │        │        │
 D₃       D₂       D₁       D₀       GND
(MSB)                      (LSB)
```

## 10.3 ADC (Analog-to-Digital Converter)
**หลักการ:** แปลงแรงดันไฟฟ้าแอนะล็อกที่มีความต่อเนื่อง ให้กลายเป็นรหัสข้อมูลดิจิทัล

### กระบวนการแปลง ADC (3 ขั้นตอน)
1. **Sampling (การสุ่มตัวอย่าง):** ตัดชิ้นส่วนของสัญญาณแอนะล็อกตามจังหวะเวลา (Clock)
   > ⚠️ **ทฤษฎีบทไนควิสต์ (Nyquist Theorem):** ความถี่ในการสุ่ม ($f_s$) ต้องมีค่ามากกว่าหรือเท่ากับ 2 เท่าของความถี่สูงสุดของสัญญาณ ($f_{max}$) มิฉะนั้นข้อมูลจะผิดเพี้ยน (Aliasing)
2. **Quantization (การจัดระดับ):** ปัดเศษค่าแรงดันที่สุ่มมาได้ ให้เข้ากับระดับแรงดันอ้างอิงที่ใกล้เคียงที่สุด (ขั้นบันได) เกิดเป็น **Quantization Error**
3. **Encoding (การเข้ารหัส):** นำระดับขั้นบันไดที่ได้มาแปลงเป็นตัวเลขฐานสอง (Binary Code)

### ชนิดของสถาปัตยกรรม ADC ที่นิยมใช้

| ชนิด (Architecture) | หลักการทำงาน | ความเร็ว | ความแม่นยำ (Resolution) | เหมาะสำหรับ |
|:---|:---|:---:|:---:|:---|
| **Flash ADC** | ใช้วงจรเปรียบเทียบ (Comparator) $2^n-1$ ตัว ทำงานพร้อมกันแบบขนาน | **เร็วที่สุด** ✅ | ต่ำ (มักไม่เกิน 8-bit) | เครื่องมือวัดความถี่สูง, สัญญาณวิดีโอ, ออสซิลโลสโคป |
| **SAR (Successive Approximation)** | สุ่มเดาค่าทีละบิตจาก MSB ไป LSB (เหมือนการเล่นทายใจตัวเลขแบบแบ่งครึ่ง) | ปานกลาง | สูง (10 ถึง 16-bit) | **ไมโครคอนโทรลเลอร์ (เช่น Arduino)**, งานเก็บข้อมูลทั่วไป |
| **Sigma-Delta ($\Sigma-\Delta$)** | สุ่มข้อมูลด้วยความถี่สูงเกินจริง (Oversampling) แล้วใช้ดิจิทัลฟิลเตอร์กรอง | ช้า | **สูงที่สุด** ✅ (16 ถึง 24-bit+) | การบันทึกเสียง (Audio), เครื่องชั่งน้ำหนักดิจิทัลที่ต้องการความละเอียดสูงมาก |

## 10.4 การเชื่อมต่อทางไฟฟ้า (Electrical Interfacing)

การนำลอจิกเกตไปต่อกับอุปกรณ์ภายนอก ต้องคำนึงถึงพารามิเตอร์ทางไฟฟ้าเสมอ:

### 10.4.1 การเชื่อมต่อลอจิกต่างระดับแรงดัน (Level Shifting)
ปัจจุบันชิปดิจิทัลมีหลายมาตรฐานแรงดัน เช่น TTL (5V) และ CMOS สมัยใหม่ (3.3V หรือ 1.8V)
- **3.3V Output $\rightarrow$ 5V Input:** มักต่อตรงได้เลย (เพราะลอจิก 1 ของ 5V TTL ต้องการแค่ 2.0V ขึ้นไป)
- **5V Output $\rightarrow$ 3.3V Input:** **ห้ามต่อตรงเด็ดขาด** (ชิป 3.3V อาจพัง) ต้องใช้วงจรแบ่งแรงดัน (Voltage Divider) หรือชิป Level Shifter (เช่น 74LVC245)

### 10.4.2 การใช้ Pull-up และ Pull-down Resistor
อินพุตของลอจิกเกต (โดยเฉพาะตระกูล CMOS) มีความต้านทานสูงมาก หากปล่อยขาลอยไว้ (Floating) สัญญาณจะแกว่งไปมา (0 บ้าง 1 บ้าง) ตามคลื่นแม่เหล็กไฟฟ้าในอากาศ
- **Pull-up:** ต่อตัวต้านทาน ($10 k\Omega$) คล่อมระหว่างขา Input กับ Vcc (เพื่อให้สถานะปกติเป็น 1)
- **Pull-down:** ต่อตัวต้านทาน ($10 k\Omega$) คล่อมระหว่างขา Input กับ GND (เพื่อให้สถานะปกติเป็น 0)
- *มักใช้ร่วมกับสวิตช์ปุ่มกด (Push Button)*

### 10.4.3 การขับโหลดกระแสสูง (Driving Loads)
ขา Output ของลอจิกเกตทั่วไป (เช่น 74LSxx) จ่ายกระแสได้น้อยมาก (ไม่เกิน $20 mA$) ไม่พอสำหรับการหมุนมอเตอร์ หรือดึงรีเลย์
- **วิธีแก้:** ต้องใช้อุปกรณ์ขยายกระแส เช่น **Transistor (NPN)** หรือ **MOSFET** มาต่อเป็นสวิตช์อิเล็กทรอนิกส์

## 10.5 การใช้งาน Tinkercad Circuits สำหรับออกแบบลอจิก

**Tinkercad Circuits** เป็นเครื่องมือออนไลน์ฟรีจาก Autodesk ที่จำลองวงจรอิเล็กทรอนิกส์และลอจิกเกตได้เสมือนต่อบนบอร์ดทดลอง (Breadboard) จริง

### 10.5.1 การเตรียมความพร้อมก่อนต่อวงจร (Best Practices)
1. **จัดระเบียบสายไฟ:** 
   - ใช้สายไฟ **สีแดง สำหรับ VCC (5V)** และ **สีดำ สำหรับ GND** เสมอ
   - จัดสายให้อยู่ในแนวราบและแนวดิ่ง ไม่โยงสายพาดข้ามตัวไอซี (คลิกที่สายเพื่อสร้างจุดหักมุม)
2. **การต่อไฟให้ชิป 74xx:** ชิปลอจิกส่วนใหญ่ ขาบนซ้ายสุดคือ VCC (ขา 14 หรือ 16) และขวาล่างสุดคือ GND (ขา 7 หรือ 8) **ต้องต่อไฟ 2 ขานี้ก่อนเสมอ วงจรถึงจะทำงาน**
3. **การป้องกัน LED ขาด:** เมื่อต่อ LED ต้องต่ออนุกรมกับตัวต้านทาน (แนะนำค่า $220\Omega$ ถึง $1 k\Omega$) เพื่อจำกัดกระแสไฟ
4. **ขาที่ไม่ได้ใช้งาน (Unused Inputs):** ในโลกจริงและใน Tinkercad ควรต่อขา Input ของเกตที่ไม่ได้ใช้งานลง GND ป้องกันสัญญาณรบกวน

### 10.5.2 ขั้นตอนการตรวจสอบข้อผิดพลาด (Troubleshooting)
หากต่อวงจรแล้วไฟไม่ติด หรือผลลัพธ์ไม่ตรงตาม Truth Table ให้เช็คตามลำดับนี้:
1. **Power:** ต่อ VCC และ GND เข้าชิปทุกตัวหรือยัง?
2. **Orientation:** วางชิปกลับหัวหรือไม่? (สังเกต "รอยบาก" หรือ "จุด" ต้องอยู่ทางซ้ายมือ)
3. **Floating Inputs:** มีขา Input ลอยอยู่หรือไม่? (โดยเฉพาะสวิตช์ ต่อ Pull-up/Pull-down หรือยัง?)
4. **Short Circuit:** มีการต่อสาย Output ของเกต 2 ตัวชนกันหรือไม่? (จะทำให้เกิดการลัดวงจร)

## แบบฝึกหัดท้ายบท

1. **(DAC)** ระบบเสียงดิจิทัลยุคเก่าใช้ DAC ขนาด 12-bit และมีแรงดันอ้างอิง $V_{ref} = 5V$ จงคำนวณหาค่า Step Size (ความละเอียดต่อ 1 ขั้น) และแรงดันเอาต์พุตสูงสุด ($V_{FS}$) 
2. **(ADC)** หากต้องการวัดอุณหภูมิห้องตั้งแต่ 0 ถึง 50 องศาเซลเซียส โดยต้องการให้เครื่องวัดมีความละเอียดที่สามารถแยกความแตกต่างได้ทุกๆ 0.1 องศาเซลเซียส จงคำนวณว่าต้องใช้ ADC ความละเอียดขั้นต่ำกี่บิต?
3. **(Interfacing)** ทำไมเราจึงไม่ควรนำขา Output ของชิปดิจิทัล (เช่น 7408) ไปต่อตรงเข้ากับมอเตอร์ DC 5V เพื่อสั่งให้มอเตอร์หมุน? และควรแก้ปัญหานี้อย่างไร?
4. **(Tinkercad)** จงวาดผังการต่อวงจรและระบุหมายเลขขา (Pin Diagram) ของ IC 7408 และ 7432 หากต้องการสร้างวงจร $Y = (A \cdot B) + C$
5. **(Architecture)** ระหว่างวงจรแปลงสัญญาณแบบ SAR ADC และ Flash ADC ชนิดใดที่มีความเร็วในการแปลงสัญญาณสูงกว่ากัน และเพราะเหตุใด Flash ADC จึงไม่นิยมสร้างให้มีความละเอียดเกิน 8 บิต?
