# Chapter 9: เครื่องจักรสถานะจำกัด (Finite State Machines)

## 9.1 บทนำ
**Finite State Machine (FSM)** คือแบบจำลองทางคณิตศาสตร์ที่ใช้ในการออกแบบวงจรตรรกะเชิงลำดับ (Sequential Logic Circuits) ที่มีความซับซ้อน โดยระบบจะมี "สถานะ (State)" ที่มีจำนวนจำกัด และจะเปลี่ยนจากสถานะหนึ่งไปยังอีกสถานะหนึ่ง (State Transition) ตามเงื่อนไขของอินพุตและจังหวะสัญญาณนาฬิกา (Clock)

**ส่วนประกอบหลักของ FSM:**
1. **States (สถานะ):** สภาวะของระบบในปัจจุบัน (เก็บอยู่ใน Flip-Flops)
2. **Inputs (อินพุต):** สัญญาณจากภายนอกที่มากระตุ้นการเปลี่ยนสถานะ
3. **Outputs (เอาต์พุต):** ผลลัพธ์ที่ระบบส่งออกมา
4. **Transitions (การเปลี่ยนสถานะ):** กฎที่บอกว่าถ้าระบบอยู่สถานะนี้ และได้รับอินพุตนี้ จะเปลี่ยนไปสถานะใดต่อไป

---

## 9.2 ประเภทของ FSM: Moore vs Mealy

FSM แบ่งออกเป็น 2 ประเภทหลักตามวิธีการสร้างเอาต์พุต:

### 9.2.1 Moore Machine
**เอาต์พุตขึ้นอยู่กับ "สถานะปัจจุบัน (Present State)" เพียงอย่างเดียว**
- **สัญลักษณ์ใน State Diagram:** เอาต์พุตจะถูกเขียนไว้ **ภายในวงกลม** ของสถานะ เช่น `S0 / Y=1`
- **ลักษณะการทำงาน:** เอาต์พุตจะเปลี่ยนแปลงก็ต่อเมื่อขอบสัญญาณนาฬิกา (Clock Edge) ทำให้ State เปลี่ยนแปลงเท่านั้น
- **ข้อดี:** เอาต์พุตมีความเสถียร ไม่มีสัญญาณรบกวนสั้นๆ (Glitch)
- **ข้อเสีย:** ตอบสนองต่ออินพุตช้ากว่า (ต้องรอให้ Clock ลูกถัดไปมาถึงเพื่อให้ State เปลี่ยนก่อน) และมักต้องใช้จำนวน State มากกว่า

### 9.2.2 Mealy Machine
**เอาต์พุตขึ้นอยู่กับ "สถานะปัจจุบัน" และ "อินพุตปัจจุบัน"**
- **สัญลักษณ์ใน State Diagram:** เอาต์พุตจะถูกเขียนไว้ **บนเส้นลูกศร (Transition)** คู่กับอินพุต เช่น `Input / Output`
- **ลักษณะการทำงาน:** หากอินพุตเปลี่ยน เอาต์พุตสามารถเปลี่ยนตามได้ **ทันที** โดยไม่ต้องรอ Clock ลูกถัดไป (Asynchronous Output)
- **ข้อดี:** ตอบสนองเร็วกว่า และสามารถใช้จำนวน State น้อยกว่าในการแก้ปัญหาเดียวกัน
- **ข้อเสีย:** เอาต์พุตอาจเกิด Glitch ได้ง่ายหากอินพุตมีสัญญาณรบกวนก่อนที่ขอบ Clock จะมา

## 9.3 ขั้นตอนมาตรฐานในการออกแบบ FSM (FSM Design Flow)

การแปลงโจทย์ปัญหาให้กลายเป็นวงจรฮาร์ดแวร์ มีกระบวนการ 7 ขั้นตอนดังนี้:

1. **ทำความเข้าใจข้อกำหนด (Specification):** วิเคราะห์โจทย์ว่าต้องการ Input/Output อะไร
2. **วาด State Diagram:** แปลงพฤติกรรมให้อยู่ในรูปกราฟวงกลมและลูกศร
3. **สร้าง State Table:** แปลงกราฟให้อยู่ในรูปตาราง (Present State, Input $\rightarrow$ Next State, Output)
4. **ลดรูปสถานะ (State Minimization):** ยุบรวม State ที่ทำหน้าที่ซ้ำซ้อนกันเพื่อประหยัด Flip-Flop
5. **กำหนดรหัสสถานะ (State Assignment):** แปลงชื่อ State (S0, S1) ให้เป็นรหัสไบนารี (00, 01)
6. **สร้างตารางกระตุ้น (Excitation Table):** เลือกชนิด Flip-Flop (D, JK, T) และหาค่า Input ของ FF ที่จะทำให้เกิด Next State ตามที่ต้องการ
7. **หาตรรกะสมการและวาดวงจร:** ใช้ K-Map ลดรูปสมการสำหรับ FF Inputs (Next State Logic) และสมการ Outputs

## 9.4 ตัวอย่างที่ 1: ตัวตรวจจับลำดับข้อมูล (Sequence Detector) - สไตล์ Moore

**โจทย์:** ออกแบบวงจรที่รับข้อมูลอนุกรม (Serial Input `X`) และให้เอาต์พุต `Y=1` เมื่อตรวจพบรูปแบบบิต **"11"** (หนึ่งติดกันสองตัว) โดยอนุญาตให้ตรวจจับแบบเหลื่อมทับซ้อนได้ (Overlapping) ใช้ D Flip-Flop

### ขั้นที่ 1: วาด State Diagram
กำหนดสถานะ:
- **S0:** สถานะเริ่มต้น (Reset) หรือเพิ่งรับค่า 0 (ยังไม่เจอ 1 เลย) $\rightarrow Y=0$
- **S1:** ตรวจพบ '1' ตัวแรก $\rightarrow Y=0$
- **S2:** ตรวจพบ '1' ตัวที่สอง (ครบ "11") $\rightarrow Y=1$

```text
       0             0
    ┌─────┐      ┌─────┐
    ↓     │      ↓     │
  ┌───────┐    ┌───────┐    ┌───────┐
  │  S0   │─1─→│  S1   │─1─→│  S2   │─1─┐
  │ (Y=0) │←0──│ (Y=0) │    │ (Y=1) │←┘ (ทับซ้อน: 11"1" นับเป็น 11 อีกชุด)
  └───────┘    └───────┘    └───────┘
```

### ขั้นที่ 2 & 3: State Table และ State Assignment
กำหนดรหัส: $S0 = 00, S1 = 01, S2 = 10$ (ใช้ FF 2 ตัว คือ $Q_1, Q_0$)

| Present State ($Q_1 Q_0$) | Input ($X$) | Next State ($Q_1^+ Q_0^+$) | Output ($Y$) |
|:---:|:---:|:---:|:---:|
| 00 (S0) | 0 | 00 (S0) | 0 |
| 00 (S0) | 1 | 01 (S1) | 0 |
| 01 (S1) | 0 | 00 (S0) | 0 |
| 01 (S1) | 1 | 10 (S2) | 0 |
| 10 (S2) | 0 | 00 (S0) | 1 |
| 10 (S2) | 1 | 10 (S2) | 1 |
*หมายเหตุ: State 11 (Don't care) ไม่ได้ใช้งาน*

### ขั้นที่ 4 & 5: D Flip-Flop Excitation
สำหรับ D Flip-Flop พฤติกรรมคือ $D = Q^+$ (ค่าที่ต้องการให้เป็นต่อไป คือค่าที่ต้องป้อนเข้าขา D)

### ขั้นที่ 6: ลดรูปด้วยสมการ (หา D1, D0 และ Y)
จากการดูตาราง (สามารถใช้ K-Map 3 ตัวแปรได้):
- $D_1$ จะเป็น 1 เมื่อ ($Q_1 Q_0 = 01$ และ $X=1$) หรือ ($Q_1 Q_0 = 10$ และ $X=1$)
  $\Rightarrow D_1 = Q_0X + Q_1X = X(Q_0 + Q_1)$
- $D_0$ จะเป็น 1 เมื่อ ($Q_1 Q_0 = 00$ และ $X=1$)
  $\Rightarrow D_0 = \overline{Q_1}\overline{Q_0}X$
- เอาต์พุต $Y$ ขึ้นอยู่กับ State S2 ($Q_1 Q_0 = 10$) เท่านั้น
  $\Rightarrow Y = Q_1$

## 9.5 การนำไปประยุกต์ใช้งานจริง (Practical Applications)

### ตัวอย่างที่ 2: ระบบควบคุมตู้ขายน้ำอัตโนมัติ (Vending Machine) แบบ Mealy
**ข้อกำหนด (Specs):**
- น้ำราคา 15 บาท เครื่องรับเหรียญ 5 บาท (X=0) และ 10 บาท (X=1) ปล่อยเหรียญได้ทีละ 1 เหรียญต่อ Clock
- ถ้าหยอดครบ 15 บาท (หรือเกิน) ให้ปล่อยน้ำ ($Y=1$) เครื่องไม่ทอนเงิน
- เมื่อปล่อยน้ำแล้ว ระบบจะกลับไปสถานะเริ่มต้นอัตโนมัติ

**การออกแบบ State Diagram:**
สถานะจะเก็บ "ยอดเงินสะสม" ปัจจุบัน
- **S0:** มีเงิน 0 บาท
- **S1:** มีเงิน 5 บาท
- **S2:** มีเงิน 10 บาท

*(เนื่องจากเป็น Mealy เอาต์พุตจะอยู่บนเส้นลูกศรในรูปแบบ `Input/Output`)*

```text
             0/0
         ┌────────┐
         ↓        │
      ┌────┐    ┌────┐
   ┌─→│ S0 │─0/0→│ S1 │
   │  └────┘    └────┘
   │    │ 1/0     │ 1/1 (รวม 15 บ. จ่ายน้ำ)
   │    ↓         │
   │  ┌────┐      │
   │  │ S2 │←0/0──┘
   │  └────┘
   │    │ 1/1 (รวม 20 บ. จ่ายน้ำ)
   └────┘
```

**ทำไมถึงเลือก Mealy?**
ในระบบตู้หยอดเหรียญ เมื่อลูกค้าหยอดเหรียญสุดท้ายครบ 15 บาท (เช่น มี 10 บาทอยู่แล้ว หยอดเพิ่ม 5 บาท) สัญญาณเซ็นเซอร์เหรียญ (Input) จะทำให้ตู้จ่ายน้ำออกมา **ทันที** (Output=1 บนเส้น Transition) จากนั้นใน Clock ถัดไป State จึงจะกลับไปที่ S0 เพื่อรอรับลูกค้ารายใหม่ ทำให้ระบบตอบสนองทันใจและใช้ State น้อยกว่า Moore

## 9.6 เทคนิคการเข้ารหัสสถานะ (State Encoding Techniques)

การเลือกวิธีแปลงชื่อ State เป็น Binary มีผลกระทบต่อขนาดและความเร็วของวงจรอย่างมาก

| เทคนิค | ตัวอย่าง S0 $\rightarrow$ S3 | จำนวน FF ที่ใช้ | ข้อดี / ข้อเสีย | เหมาะกับงาน |
|:---|:---:|:---:|:---|:---|
| **Binary** | 00, 01, 10, 11 | $\lceil \log_2 N \rceil$ (น้อยที่สุด) | ประหยัดฮาร์ดแวร์ แต่วงจรแก้สมการซับซ้อน | งานทั่วไปบนชิปขนาดเล็ก |
| **Gray Code** | 00, 01, 11, 10 | $\lceil \log_2 N \rceil$ (น้อยที่สุด) | เปลี่ยนบิตทีละ 1 บิต ลดโอกาสเกิด Glitch | ระบบนับ (Counters), อุปกรณ์สื่อสาร |
| **One-Hot** | 0001, 0010, 0100, 1000 | $N$ (มากที่สุด) | วงจรลอจิกแก้สมการสั้นและทำงานเร็วมาก | ชิป FPGA หรือ CPU ที่มีความเร็วสูง |

## แบบฝึกหัดท้ายบท

1. อธิบายข้อแตกต่างทางสถาปัตยกรรมระหว่างวงจร Moore Machine และ Mealy Machine (วาด Block Diagram ประกอบ)
2. หากต้องการออกแบบวงจรที่ตรวจจับลำดับ `101` แบบซ้อนทับกันไม่ได้ (Non-overlapping) จงวาด State Diagram ในรูปแบบ Moore FSM
3. ออกแบบวงจรนับขึ้น/ลง (Up/Down Counter) MOD-4 (0, 1, 2, 3) 
   - มี Input `M`: ถ้า `M=0` ให้นับขึ้น (0 $\rightarrow$ 1 $\rightarrow$ 2 $\rightarrow$ 3 $\rightarrow$ 0), ถ้า `M=1` ให้นับลง (0 $\rightarrow$ 3 $\rightarrow$ 2 $\rightarrow$ 1 $\rightarrow$ 0)
   - จงสร้าง State Table และ Excitation Table โดยใช้ **JK Flip-Flop**
4. จงใช้เทคนิค One-Hot Encoding กำหนดรหัสสถานะให้กับ FSM ที่มี 5 สถานะ จะต้องใช้ Flip-Flop ทั้งหมดกี่ตัว?
5. จากตัวอย่างวงจร Vending Machine (Mealy) ในบทเรียน จงเขียน State Table และหาสมการของ Next State Logic และ Output Logic โดยใช้ D Flip-Flop
