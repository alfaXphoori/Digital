## ส่วนที่ 1: BCD Code (15 นาที)

### แบบฝึก 1.1 — Decimal → BCD

จงแปลงเลขฐาน 10 เป็น BCD (แต่ละหลัก → 4 บิต)

| Decimal | BCD |
|:---:|:---:|
| 47 | |
| 139 | |
| 256 | |
| 908 | |

### แบบฝึก 1.2 — BCD → Decimal

| BCD | Decimal |
|:---:|:---:|
| 0011 0101 | |
| 1000 0010 0111 | |
| 0001 0000 0000 | |

> ⚠️ BCD แต่ละกลุ่ม 4 บิตต้อง ≤ 1001 (9) ถ้าเกินจะเป็นรหัสที่ไม่ valid!

### แบบฝึก 1.3 — Invalid BCD?

จงตรวจว่ารหัสต่อไปนี้เป็น BCD ที่ถูกต้องหรือไม่

| รหัส | Valid BCD? | เหตุผล |
|:---:|:---:|:---|
| 0100 0111 | | |
| 1010 0011 | | |
| 1001 1001 | | |
| 0110 1100 | | |

## ส่วนที่ 3: ASCII Code (10 นาที)

### แบบฝึก 3.1 — ข้อความ → ASCII (Hex)

จงแปลงข้อความต่อไปนี้เป็น ASCII (แสดงเป็นเลขฐาน 16)

| ตัวอักษร | ASCII (Hex) | ASCII (Binary) |
|:---:|:---:|:---:|
| 'A' | | |
| 'z' | | |

> 💡 A = 41h, a = 61h, 0 = 30h, space = 20h

## ส่วนที่ 4: Parity Bit — ตรวจจับ Error (15 นาที)

### แบบฝึก 4.1 — เพิ่ม Even Parity Bit

จงเพิ่ม Parity Bit (บิตซ้ายสุด) ให้จำนวน 1 ในข้อมูลรวมเป็น **เลขคู่**

| ข้อมูล (7 บิต) | จำนวน 1 | Parity Bit | ข้อมูลพร้อม Parity (8 บิต) |
|:---:|:---:|:---:|:---:|
| 100 1011 | | | |
| 011 0100 | | | |
| 111 1111 | | | |
| 000 0000 | | | |

### แบบฝึก 4.2 — ตรวจจับ Error

ข้อมูลต่อไปนี้ใช้ **Even Parity** จงตรวจว่ามี error หรือไม่

| ข้อมูลที่ได้รับ (8 บิต) | จำนวน 1 | มี Error? |
|:---:|:---:|:---:|
| 1 100 1010 | | |
| 0 011 0101 | | |
| 1 111 0000 | | |
| 0 110 0110 | | |
